# StockBot: 台股輿情熱度雷達 - 專案方向與需求分析

## 一、 核心目標
建立一個針對「台灣股市」的即時輿情監控系統（輿情熱度雷達），透過即時捕捉社群與新聞的討論熱度，結合市場量價變化，發掘潛在的交易機會與避開出貨陷阱。

## 二、 核心分析維度（量化交易邏輯）
1. **族群輪動與擴散效應**
   - 監控主流題材（如 AI、電力、矽光子）的資金流向。
   - 當指標股過熱時，自動掃描並提示具備相同標籤且剛起漲的「二線補漲股」。
2. **聲量與量價的「共振」與「背離」**
   - **共振（量價齊揚 + 聲量激增）**：判斷為真突破，市場共識形成。
   - **背離（高聲量 + 股價不漲/反跌）**：標記為高危險區或出貨警示。
   - **潛力（恐慌聲量 + 股價抗跌）**：發掘大盤大跌時的主力控盤潛力股。
3. **跨平台資訊時間差套利**
   - 利用資訊在不同平台（新聞/MOPS 源頭 -> PTT 專業討論 -> Threads/FB 散戶發酵）的傳播時間差，及早佈局。
4. **異常標籤組合（黑馬股發掘）**
   - 監控「冷門股代號 + 熱門新科技關鍵字」的異常配對，提早發現「老店新開」的題材初升段。

## 三、 系統資料來源設計
針對第一版 MVP（最小可行性產品），資料來源過濾與選擇如下：
*   **首要監控目標（高訊號價值、易於即時串接）**：PTT Stock 股板、公開資訊觀測站 (MOPS)、主流財經新聞（鉅亨網、經濟日報等）。
*   **次要/進階監控目標（散戶情緒發酵）**：Threads、社群論壇。

## 四、 V1 系統資料處理架構：混合模式 (Hybrid Architecture)
為了平衡「精準度」與「發掘新題材的能力」，採用以下策略：
1.  **核心防線：由上而下（Top-Down - 靜態字典匹配）**
    *   建立基礎「股票白名單」與「熱門概念標籤字典」（如：台股市值前 300 大、熱門中小型股、主流題材）。
    *   爬蟲文章只要比對命中白名單，即刻計算熱度並觸發警示，確保高勝率與低雜訊。
2.  **驚喜探針：由下而上（Bottom-Up - 動態關鍵字萃取）**
    *   不對全網垃圾訊息做盲搜，**只針對已經命中白名單的熱門文章**，進行輕量級 NLP 或 LLM 分析。
    *   萃取文章中與該股票高度關聯的「未知新名詞」，將其列為「新 Tag 候選人」詢問使用者是否加入字典，解決發掘新題材的需求。

## 五、 系統技術藍圖規劃建議 (C# / .NET 生態系)
1.  **資料收集層 (Data Ingestion)**
    *   **技術方向**：.NET Worker Service / Hangfire 排程。
    *   **實作目標**：建立穩定的 PTT/新聞 爬蟲或 WebSocket 串接。
2.  **處理與關聯層 (Processing & NLP)**
    *   **技術方向**：高效字串比對演算法 (如 Aho-Corasick) 搭配 Semantic Kernel 呼叫 LLM API。
    *   **實作目標**：建立動態關聯字典 (Knowledge Graph)、股票實體識別 (NER)、文本情緒判定。
3.  **儲存與量化層 (Storage & Time-Series)**
    *   **技術方向**：直接採用 **InfluxDB**。
    *   **實作目標**：儲存時序熱度資料與處理向量 (Vector) 等相關運算，作為量價背離與熱度分析的基礎。
4.  **警報輸出層 (Alerting)**
    *   **技術方向**：**Telegram Bot API**。
    *   **實作目標**：透過 Telegram 機器人/頻道，將即時的「共振」、「出貨背離」等量價訊號推播至終端。
